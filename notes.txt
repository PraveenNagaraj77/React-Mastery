What is React?

React is an open-source JavaScript library developed by Facebook for building reusable and efficient user interfaces. It follows a component-based architecture and uses a Virtual DOM to optimize UI updates and improve performance.

What is the difference between SPA and MPA?

A Multi Page Application reloads the entire page whenever the user navigates to another page. A Single Page Application loads the application once and dynamically updates only the required parts of the UI without reloading the entire page. React applications are generally SPAs, which provide a faster and smoother user experience.

What is Virtual DOM?

Virtual DOM is a lightweight JavaScript representation of the real DOM. Whenever state or props change, React creates a new Virtual DOM tree and compares it with the previous one using a diffing algorithm. React then updates only the changed elements in the real DOM through a process called reconciliation, improving performance and reducing unnecessary DOM operations.


Is Virtual DOM Faster Than Real DOM?

No. Virtual DOM itself is not faster than the real DOM. The benefit comes from minimizing expensive real DOM updates. React performs calculations in memory using the Virtual DOM and updates only the necessary parts of the real DOM.


What is JSX?

JSX stands for JavaScript XML. It is a syntax extension for JavaScript that allows developers to write HTML-like code inside JavaScript. JSX improves readability and developer experience. Browsers cannot understand JSX directly, so it is converted into JavaScript using Babel.

What is a Component?

A component is an independent and reusable piece of UI in React. Components help divide the application into smaller manageable units, improving code reusability, maintainability, and scalability.

Why are Components Important?

Components allow developers to build complex user interfaces by composing smaller reusable pieces. This reduces code duplication and makes applications easier to maintain.

What are Props?

Props are read-only inputs passed from a parent component to a child component. They are used to transfer data and make components reusable and dynamic.

Can a Child Modify Props?

No. Props are immutable. A child component can read props but cannot modify them. Data flow in React is unidirectional from parent to child.

Why are Props Important?

Props help create reusable and configurable components by allowing dynamic data to be passed from parent components to child components.

What is a State?

State is data managed by React that can change over time.State is a built-in React mechanism used to store and manage dynamic data within a component. Whenever state changes, React re-renders the component and updates the UI accordingly.

What is useState?

useState is a React Hook that allows functional components to manage state. It returns the current state value and a setter function used to update that state.


Difference Between State and Props?

Props are read-only values passed from parent to child components, whereas state is managed within a component and can change over time using setter functions.

Why are you using prev => prev + 1 instead of cart + 1?

Because the new state depends on the previous state. Using a functional update ensures React always uses the latest state value, especially when updates are batched.

What is Event Handling in React?

Event handling in React is the mechanism used to respond to user interactions such as clicks, input changes, and form submissions. React uses synthetic events and camelCase event names like onClick and onChange.

Why do we use preventDefault()?

preventDefault() prevents the browser's default behavior. In React forms, it is commonly used to stop page refreshes during form submission.

What is a Controlled Component?

A controlled component is a form element whose value is managed by React state. The displayed value is controlled through the value prop, and updates are handled using event handlers such as onChange.

Why Use Controlled Components?

Controlled components provide better control over form data, make validation easier, and ensure React remains the single source of truth for UI state.

Why do we use value={state} in form inputs?

Using the value prop makes the input a controlled component. React state becomes the single source of truth, allowing easier validation, form handling, and UI updates.\


What is Form Validation?

Form validation is the process of verifying user input before form submission to ensure that data meets predefined rules such as required fields, valid email format, or password length.

What is useRef?

useRef is a React Hook that provides a mutable reference object. It is commonly used to access DOM elements directly and store values that persist across renders without causing re-renders.

Difference Between useRef and useState?

useState triggers component re-renders when updated, whereas updating a useRef value does not trigger a re-render. useState is used for UI data, while useRef is used for DOM references and mutable values.

What is React Hook Form?

React Hook Form is a lightweight library for managing forms in React. It provides built-in validation, better performance by minimizing re-renders, and simpler code compared to managing forms manually with useState.


What does register() do?

register() connects an input field to React Hook Form so it can track its value, apply validation rules, and include it in the submitted form data.

What is handleSubmit()?

handleSubmit() validates the form first. If validation succeeds, it calls the provided callback with the form data.

What is errors?

errors is an object containing validation errors for each registered field. It is used to display user-friendly validation messages.

What is useEffect?

useEffect is a React Hook that lets you perform side effects in a component. A side effect is anything that happens outside of rendering the UI.

What is useEffect?


useEffect is a React Hook used to perform side effects in functional components. It runs after the component renders and can be configured to run once, on every render, or only when specified dependencies change.

Examples:

Fetching data from an API
Setting up timers
Updating the document title
Adding event listeners
Accessing local storage

What is the dependency array?


The dependency array controls when a useEffect runs. An empty array ([]) runs the effect only once, no array runs it after every render, and listing dependencies runs it whenever those values change.


What is a Cleanup Function?

A cleanup function is a function returned from useEffect. React executes it before the effect runs again due to dependency changes and when the component unmounts. It is commonly used to clear timers, remove event listeners, and cancel subscriptions.


Why is Cleanup Important?

Cleanup prevents memory leaks and unwanted side effects by releasing resources such as timers, event listeners, and subscriptions when they are no longer needed.


Why do we remove event listeners in useEffect?

We remove event listeners in the cleanup function to prevent memory leaks and duplicate event handlers when the component unmounts or the effect is re-executed.

Why should we avoid anonymous functions with addEventListener?

removeEventListener requires the exact same function reference that was passed to addEventListener. Anonymous functions create a new reference each time, making them impossible to remove correctly.


Why are API calls usually placed inside useEffect?

API calls are side effects. Placing them inside useEffect ensures they run after the component renders and can be controlled using the dependency array to avoid unnecessary or repeated requests.

Why use an empty dependency array for fetching data?

An empty dependency array ([]) makes the effect run only once after the initial render, preventing repeated API calls on every re-render.

What is the purpose of loading state?

It lets the UI show a loading indicator while waiting for asynchronous operations like API requests to complete.

Why use try...catch...finally?

try → executes the API call
catch → handles any errors
finally → runs regardless of success or failure, making it ideal for stopping loading indicators.

Why check response.ok?

fetch() only rejects on network errors. HTTP errors like 404 or 500 do not automatically throw an exception, so checking response.ok ensures those responses are handled properly.

Can a component have multiple useEffects?

Yes. A React component can have multiple useEffect hooks. Each one should ideally handle a single side effect, improving readability, separation of concerns, and maintainability.

In what order are multiple useEffects executed?

They execute in the order they are declared in the component, after React has rendered the UI.

Why split logic into multiple useEffects?

To separate unrelated side effects. This makes the component easier to understand, maintain, and debug.

What is a stale closure in React?

A stale closure occurs when a function inside a useEffect or asynchronous callback captures an old state or prop value from the render in which it was created, causing it to use outdated data even after the component re-renders.

Does updating a ref trigger a re-render?

No. Updating ref.current does not trigger a re-render.

Why doesn't updating ref.current update the UI?

Because React only re-renders a component when its state or props change. Updating ref.current changes the stored value, but React does not track ref updates for rendering, so the component does not re-render.

“Updating ref.current does not cause a re-render, but the updated value will appear in the UI during the next re-render triggered by state or props changes.”


What is useMemo?

useMemo is a React Hook used to memoize the result of an expensive calculation. It caches the computed value and recalculates it only when one of its dependencies changes. This helps avoid unnecessary computations during component re-renders, improving performance.

What is Memoization?

Memoization is an optimization technique where the result of an expensive function is stored (cached). If the same inputs occur again, the cached result is returned instead of executing the function again.

Explain useMemo in two minutes (Interview)

"useMemo is a performance optimization hook in React. It memoizes the result of an expensive computation and recalculates it only when one of its dependencies changes. This avoids unnecessary calculations during component re-renders. For example, in an employee search application, filtering a large employee list can be expensive. By wrapping the filtering logic in useMemo, React reuses the previously computed filtered list unless the search input changes. However, useMemo does not prevent component re-renders; it only memoizes the computed value."


What is React.memo?


React.memo is a Higher-Order Component that memoizes a component. It prevents unnecessary re-renders by comparing the current props with the previous props using shallow comparison. If the props are unchanged, React skips re-rendering the component.

Why is useCallback often used with React.memo?

"React.memo skips re-rendering only when a component's props remain the same. Function props are recreated on every parent render, so their references change even if the implementation is identical. useCallback memoizes the function reference, allowing React.memo to see the same prop and skip unnecessary child re-renders."


useCallback memoizes a function so that React returns the same function reference between renders (unless its dependencies change). When this memoized function is passed as a prop to a child component wrapped with React.memo, React compares the previous and current props. Since the function reference remains the same, React.memo determines that the props haven't changed and skips the unnecessary re-render of the child component

Why do we use useCallback?

"useCallback is a React Hook that memoizes a function. It returns the same function instance across renders unless its dependencies change. It is commonly used with React.memo to prevent unnecessary re-rendering of child components when functions are passed as props."

What is a Custom Hook?

A Custom Hook is a JavaScript function whose name starts with use and that uses one or more React Hooks to encapsulate and reuse stateful logic across multiple components.


What is Prop Drilling?

Prop Drilling is the process of passing data from a parent component to a deeply nested child component through intermediate components that don't actually use the data. It can make the code harder to maintain, and Context API helps solve this problem.

React Router

React Router is a client-side routing library for React that enables navigation between different pages without reloading the browser. It creates a Single Page Application (SPA) experience by rendering different components based on the URL while keeping the application loaded in the browser. React Router improves performance and provides a seamless user experience.

BrowserRouter

BrowserRouter is the top-level router component that enables routing in a React application. It uses the HTML5 History API to manage navigation and keeps the UI synchronized with the browser URL. It wraps the entire application so that routing features are available throughout the app.

Routes

The Routes component acts as a container for all the route definitions. It checks the current URL and renders the first matching Route. It replaces the older Switch component from React Router v5 and ensures only one matching route is displayed.

Route

A Route maps a URL path to a React component. When the browser URL matches the specified path, the component provided in the element prop is rendered. Routes define the navigation structure of the application.


Link

The Link component is used for navigation between pages without refreshing the browser. Unlike the HTML <a> tag, Link performs client-side navigation, making the application faster and preserving the React state.


NavLink

NavLink is an enhanced version of Link that automatically detects the active route. It is commonly used for navigation menus because it allows active links to be styled differently.

Layout Routes

Layout Routes are used when multiple pages share common UI elements such as a navbar, sidebar, or footer. Instead of repeating these components on every page, they are placed inside a layout component, and child routes are rendered within it using Outlet.

Outlet

The Outlet component is a placeholder where nested child routes are rendered. It allows a parent component to display common UI while dynamically rendering the content of child pages.

useNavigate

useNavigate is a React Router hook used for programmatic navigation. Instead of navigating through user clicks on links, it allows navigation inside JavaScript functions, such as after login, logout, or successful form submission.


Navigate

The Navigate component is used to redirect users while rendering a component. It is commonly used for authentication and protected routes. Unlike useNavigate, it is a component rather than a hook.


Route Parameters

Route Parameters are dynamic values passed through the URL. They allow a single route to display different data based on the parameter value.



useParams

useParams is a React Router hook used to access dynamic route parameters from the URL. It returns an object containing all route parameters, making it easy to fetch or display data based on the current URL.


Nested Routes

Nested Routes allow child pages to be grouped under a parent route. They are useful for dashboards, admin panels, and applications where multiple related pages share a common layout. The parent component renders shared UI, while child routes are displayed using Outlet.


Protected Routes

Protected Routes restrict access to specific pages based on authentication or authorization. If the user is authenticated, the requested page is rendered. Otherwise, the user is redirected to the login page. Protected routes are commonly implemented using Context API or Redux together with the Navigate component.

useSearchParams

useSearchParams is a React Router hook used to read and update query parameters in the URL. It is commonly used for search, filtering, sorting, and pagination.

404 Route

A 404 Route is used to handle invalid or unknown URLs. It is typically defined using the wildcard (*) path and displays a "Page Not Found" component when no other route matches.

Interview Summary (One-Liner)

React Router is a client-side routing library for React that enables navigation between components without reloading the page. It provides features such as route matching, nested routing, dynamic route parameters, protected routes, query parameters, layouts, and programmatic navigation, making it easy to build scalable Single Page Applications (SPAs).



What  is Redux?

Redux is a predictable state management library that stores the application's global state in a single centralized store. Components can read data from the store and update it by dispatching actions, making state management more organized, scalable, and easier to debug.

When would you choose Context API over Redux Toolkit?

I use Context API for simple global state like authentication, theme, or language because it's built into React and easy to set up. For applications with complex shared state, frequent updates, or multiple features such as shopping carts, products, orders, and user management, I prefer Redux Toolkit because it provides centralized state management, better scalability, predictable updates, and excellent debugging through Redux DevTools.

A Redux Store is a centralized object that holds the entire application's global state. It acts as the single source of truth, allowing components to read and update shared state in a predictable manner.


Why do we need a Store?

The Store provides a centralized location to manage the application's global state. Instead of maintaining shared state in multiple components or passing props through several levels, components can directly access and update the store, making state management predictable, scalable, and easier to debug.

What is slice?

A Slice is a feature-specific portion of the Redux Store that contains the initial state, reducers, and automatically generated actions for that feature. It helps organize Redux logic into modular and maintainable units.

What is initialState?

initialState defines the default state of a slice before any actions are dispatched.

What is a reducer?

A reducer is a function that receives the current state and an action, then returns the updated state based on that action.

Why can we mutate state in Redux Toolkit?

Redux Toolkit uses the Immer library internally. It allows us to write mutating logic like state.items.push(), but it safely produces a new immutable state behind the scenes.

What is an Action?

An action is a plain JavaScript object that describes what happened. It contains a type property and optionally a payload that carries the data required to update the state.

What is an Action Creator?

An Action Creator is a function that creates and returns an action object. In Redux Toolkit, action creators are generated automatically by createSlice().

What is UseDispatch?

UseDispatch is a React Redux hook that returns the dispatch function, which is used to send actions to the Redux Store. Dispatching an action tells Redux that something happened and allows reducers to update the state.

What is UseSelector?

useSelector is a React Redux hook used to read data from the Redux Store. It subscribes the component to the selected state, and the component automatically re-renders whenever that selected state changes.


What is createAsyncThunk?

createAsyncThunk is a Redux Toolkit utility used to handle asynchronous operations such as API calls. It automatically dispatches pending, fulfilled, and rejected actions, making it easy to manage loading, success, and error states.
