What is React?

React is an open-source JavaScript library developed by Facebook for building reusable and efficient user interfaces. It follows a component-based architecture and uses a Virtual DOM to optimize UI updates and improve performance.

What is the difference between SPA and MPA?

A Multi Page Application reloads the entire page whenever the user navigates to another page. A Single Page Application loads the application once and dynamically updates only the required parts of the UI without reloading the entire page. React applications are generally SPAs, which provide a faster and smoother user experience.

What is Virtual DOM?

Virtual DOM is a lightweight JavaScript representation of the real DOM. Whenever state or props change, React creates a new Virtual DOM tree and compares it with the previous one using a diffing algorithm. React then updates only the changed elements in the real DOM through a process called reconciliation, improving performance and reducing unnecessary DOM operations.


Is Virtual DOM Faster Than Real DOM?

No. Virtual DOM itself is not faster than the real DOM. The benefit comes from minimizing expensive real DOM updates. React performs calculations in memory using the Virtual DOM and updates only the necessary parts of the real DOM.


What is JSX?

JSX stands for JavaScript XML. It is a syntax extension for JavaScript that allows developers to write HTML-like code inside JavaScript. JSX improves readability and developer experience. Browsers cannot understand JSX directly, so it is converted into JavaScript using Babel.

What is a Component?

A component is an independent and reusable piece of UI in React. Components help divide the application into smaller manageable units, improving code reusability, maintainability, and scalability.

Why are Components Important?

Components allow developers to build complex user interfaces by composing smaller reusable pieces. This reduces code duplication and makes applications easier to maintain.

What are Props?

Props are read-only inputs passed from a parent component to a child component. They are used to transfer data and make components reusable and dynamic.

Can a Child Modify Props?

No. Props are immutable. A child component can read props but cannot modify them. Data flow in React is unidirectional from parent to child.

Why are Props Important?

Props help create reusable and configurable components by allowing dynamic data to be passed from parent components to child components.

What is a State?

State is data managed by React that can change over time.State is a built-in React mechanism used to store and manage dynamic data within a component. Whenever state changes, React re-renders the component and updates the UI accordingly.

What is useState?

useState is a React Hook that allows functional components to manage state. It returns the current state value and a setter function used to update that state.


Difference Between State and Props?

Props are read-only values passed from parent to child components, whereas state is managed within a component and can change over time using setter functions.

Why are you using prev => prev + 1 instead of cart + 1?

Because the new state depends on the previous state. Using a functional update ensures React always uses the latest state value, especially when updates are batched.

What is Event Handling in React?

Event handling in React is the mechanism used to respond to user interactions such as clicks, input changes, and form submissions. React uses synthetic events and camelCase event names like onClick and onChange.

Why do we use preventDefault()?

preventDefault() prevents the browser's default behavior. In React forms, it is commonly used to stop page refreshes during form submission.

What is a Controlled Component?

A controlled component is a form element whose value is managed by React state. The displayed value is controlled through the value prop, and updates are handled using event handlers such as onChange.

Why Use Controlled Components?

Controlled components provide better control over form data, make validation easier, and ensure React remains the single source of truth for UI state.

Why do we use value={state} in form inputs?

Using the value prop makes the input a controlled component. React state becomes the single source of truth, allowing easier validation, form handling, and UI updates.\


What is Form Validation?

Form validation is the process of verifying user input before form submission to ensure that data meets predefined rules such as required fields, valid email format, or password length.

What is useRef?

useRef is a React Hook that provides a mutable reference object. It is commonly used to access DOM elements directly and store values that persist across renders without causing re-renders.

Difference Between useRef and useState?

useState triggers component re-renders when updated, whereas updating a useRef value does not trigger a re-render. useState is used for UI data, while useRef is used for DOM references and mutable values.

What is React Hook Form?

React Hook Form is a lightweight library for managing forms in React. It provides built-in validation, better performance by minimizing re-renders, and simpler code compared to managing forms manually with useState.


What does register() do?

register() connects an input field to React Hook Form so it can track its value, apply validation rules, and include it in the submitted form data.

What is handleSubmit()?

handleSubmit() validates the form first. If validation succeeds, it calls the provided callback with the form data.

What is errors?

errors is an object containing validation errors for each registered field. It is used to display user-friendly validation messages.

What is useEffect?

useEffect is a React Hook that lets you perform side effects in a component. A side effect is anything that happens outside of rendering the UI.

What is useEffect?


useEffect is a React Hook used to perform side effects in functional components. It runs after the component renders and can be configured to run once, on every render, or only when specified dependencies change.

Examples:

Fetching data from an API
Setting up timers
Updating the document title
Adding event listeners
Accessing local storage

What is the dependency array?


The dependency array controls when a useEffect runs. An empty array ([]) runs the effect only once, no array runs it after every render, and listing dependencies runs it whenever those values change.


What is a Cleanup Function?

A cleanup function is a function returned from useEffect. React executes it before the effect runs again due to dependency changes and when the component unmounts. It is commonly used to clear timers, remove event listeners, and cancel subscriptions.


Why is Cleanup Important?

Cleanup prevents memory leaks and unwanted side effects by releasing resources such as timers, event listeners, and subscriptions when they are no longer needed.


Why do we remove event listeners in useEffect?

We remove event listeners in the cleanup function to prevent memory leaks and duplicate event handlers when the component unmounts or the effect is re-executed.

Why should we avoid anonymous functions with addEventListener?

removeEventListener requires the exact same function reference that was passed to addEventListener. Anonymous functions create a new reference each time, making them impossible to remove correctly.


Why are API calls usually placed inside useEffect?

API calls are side effects. Placing them inside useEffect ensures they run after the component renders and can be controlled using the dependency array to avoid unnecessary or repeated requests.

Why use an empty dependency array for fetching data?

An empty dependency array ([]) makes the effect run only once after the initial render, preventing repeated API calls on every re-render.

What is the purpose of loading state?

It lets the UI show a loading indicator while waiting for asynchronous operations like API requests to complete.

Why use try...catch...finally?

try → executes the API call
catch → handles any errors
finally → runs regardless of success or failure, making it ideal for stopping loading indicators.

Why check response.ok?

fetch() only rejects on network errors. HTTP errors like 404 or 500 do not automatically throw an exception, so checking response.ok ensures those responses are handled properly.

Can a component have multiple useEffects?

Yes. A React component can have multiple useEffect hooks. Each one should ideally handle a single side effect, improving readability, separation of concerns, and maintainability.

In what order are multiple useEffects executed?

They execute in the order they are declared in the component, after React has rendered the UI.

Why split logic into multiple useEffects?

To separate unrelated side effects. This makes the component easier to understand, maintain, and debug.

What is a stale closure in React?

A stale closure occurs when a function inside a useEffect or asynchronous callback captures an old state or prop value from the render in which it was created, causing it to use outdated data even after the component re-renders.
